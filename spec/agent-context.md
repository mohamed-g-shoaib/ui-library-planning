## Hard Rules

### Never Do
- `asChild` ‚Äî Base UI uses the `render` prop instead
- `import * as React` ‚Äî use named imports only
- `export default function Component` ‚Äî named exports only, always
- `text-gray-500`, `bg-zinc-800`, or any Tailwind palette color ‚Äî semantic tokens only
- `motion/react` inside distributable registry components ‚Äî CSS animations only there
- `"use client"` on stateless/server-renderable components
- Edit `__index__.tsx` manually ‚Äî it is auto-generated by `build-registry.ts`
- Mix animation approaches: pick CSS presets OR Motion, never both in the same component
- `group`/`group-hover:` when a `data-[state=x]` selector or `has()` works
- Invent Base UI component names ‚Äî always verify at https://base-ui.com/react/components

### Always Do
- `data-slot="[part-name]"` on every compound sub-part
- `tv()` from `tailwind-variants` for all variant logic
- Colocate types with components: `export interface ButtonProps {}`
- Named exports: `export { Button, type ButtonProps }`
- Static `cssAnimationPresets` / `cssTransitionPresets` objects **outside** the component function
- `will-change` + `transform-gpu` on all GPU-animated elements
- `reduceMotion` prop on every animated component that accepts presets
- `aria-hidden="true"` on decorative icons
- `aria-label` on icon-only interactive elements
- `pointer-coarse:after:min-h-11 min-w-11` on all touch-target buttons
- OKLCH for all new color token definitions

## Component Skeleton

Every registry component follows this exact skeleton:

```tsx

import { tv, type VariantProps } from 'tailwind-variants';
import { cn } from '@/lib/classes';

  fade: [
    '[transition-property:opacity,height] [will-change:opacity,height]',
    'data-starting-style:opacity-0 data-ending-style:opacity-0',
  ],
  // ...etc
} as const;

const cssTransitionPresets = {
  outQuad: 'duration-[0.25s] ease-[cubic-bezier(0.25,0.46,0.45,0.94)]',
  overshootOut: 'duration-[0.35s] ease-[cubic-bezier(0.175,0.885,0.32,1.275)]',
  // ...etc
} as const;

const componentVariants = tv({
  base: 'base-classes',
  variants: {
    variant: { default: '...', destructive: '...' },

export interface ComponentProps
  extends SomePrimitive.Root.Props,
    VariantProps<typeof componentVariants> {
  animationPreset?: CSSAnimationPreset;
  transitionPreset?: CSSTransitionPreset;
  reduceMotion?: boolean;
}

// ‚îÄ‚îÄ‚îÄ Component ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ

function Component({
  variant,
  size,
  radius,
  animationPreset = 'fade',
  transitionPreset = 'outQuad',
  reduceMotion = false,
  className,
  ...props
}: ComponentProps) {
  return (
    <SomePrimitive.Root
      data-slot="component"
      className={cn(componentVariants({ variant, size, radius }), className)}
      {...props}
    />
  );
}

export { Component, type ComponentProps };
```

## The Animation System (Core Pattern)

This is the most important architectural pattern in Dusk UI. Read it carefully.

### How it works

Base UI manages open/closed lifecycle and writes `data-starting-style` / `data-ending-style` attributes onto animated elements. CSS `@starting-style` natively hooks into these. Tailwind CSS v4's `data-starting-style:` variant makes these targetable as utility classes.

**No JavaScript animation. No layout thrashing. Zero runtime cost.**

### The Two Objects

Every animated component exposes **two props**: `animationPreset` and `transitionPreset`.

```tsx
// ‚îÄ‚îÄ Animation presets (WHAT moves) ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
const cssAnimationPresets = {
  none:             'transition-none',
  fade:             ['[transition-property:opacity,height] [will-change:opacity,height]',
                     'data-starting-style:opacity-0 data-ending-style:opacity-0 data-starting-style:h-0 data-ending-style:h-0'],
  scale:            ['[transition-property:scale,opacity,height] [will-change:scale,opacity,height] origin-top',
                     'data-starting-style:scale-95 data-starting-style:opacity-0 data-ending-style:opacity-0 data-ending-style:scale-95'],
  slide:            ['[transition-property:translate,opacity,height] [will-change:translate,opacity,height]',
                     'data-starting-style:opacity-0 data-starting-style:translate-y-2 data-ending-style:translate-y-2 data-ending-style:opacity-0'],
  topSlide:         '...', bottomSlide: '...', leftSlide: '...', rightSlide: '...',
  topFlip:          '...', bottomFlip: '...', leftFlip: '...', rightFlip: '...',
  wipe:             '...', // clip-path curtain reveal
  perspective:      '...', // rotateX(90deg) + perspective(1000px)
  perspectiveBlur:  '...', // perspective + blur(9px)
} as const;

// ‚îÄ‚îÄ Transition presets (HOW it moves ‚Äî easing + duration) ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
const cssTransitionPresets = {
  // Expo
  inExpo:       'duration-[0.35s] ease-[cubic-bezier(0.95,0.05,0.795,0.035)]',
  outExpo:      'duration-[0.35s] ease-[cubic-bezier(0.19,1,0.22,1)]',
  // Anticipate / Overshoot
  anticipate:   'duration-[0.35s] ease-[cubic-bezier(1,-0.4,0.35,0.95)]',
  overshootOut: 'duration-[0.35s] ease-[cubic-bezier(0.175,0.885,0.32,1.275)]',
  swiftOut:     'duration-[0.35s] ease-[cubic-bezier(0.175,0.885,0.32,1.1)]',
  snappyOut:    'duration-[0.35s] ease-[cubic-bezier(0.19,1,0.22,1)]',
  // Quad
  inQuad:       'duration-[0.35s] ease-[cubic-bezier(0.55,0.085,0.68,0.53)]',
  outQuad:      'duration-[0.25s] ease-[cubic-bezier(0.25,0.46,0.45,0.94)]',
  inOutQuad:    'duration-[0.32s] ease-[cubic-bezier(0.455,0.03,0.515,0.955)]',
  // Cubic
  inCubic:      'duration-[0.35s] ease-[cubic-bezier(0.55,0.055,0.675,0.19)]',
  outCubic:     'duration-[0.35s] ease-[cubic-bezier(0.215,0.61,0.355,1)]',
  inOutCubic:   'duration-[0.35s] ease-[cubic-bezier(0.645,0.045,0.355,1)]',
  // Quart, Quint, Circ (6 more each) ...
  inOutBase:    'duration-[0.35s] ease-[cubic-bezier(0.25,0.1,0.25,1)]',
} as const;
```

### Usage

```tsx

<Accordion animationPreset="perspectiveBlur" transitionPreset="outCubic">
<Sheet animationPreset="rightSlide" transitionPreset="swiftOut">

// reduceMotion ‚Äî always supported

```

### When to Use Motion v12

Motion is **only** acceptable in these cases (doc site / blocks, never in `registry/ui/`):

| Use case | Pattern |
|---|---|
| Tab content fade | `AnimatePresence` + `motion.div` with opacity |
| Layout shift animation | `<motion.div layout>` |
| Staggered list | `staggerChildren` in `variants` |
| Spring physics | `type: 'spring'` in `transition` |
| Drag to dismiss | `drag` + `dragConstraints` |
| Rolling/ticker text | `motion.div` with `x` keyframes |

## `tailwind-variants` Slot Pattern

Use `tv()` slots for compound components where multiple elements must be styled together.

```tsx
// Example: InputGroup with slots
const inputGroupVariants = tv({
  slots: {
    root:    'flex items-center rounded-md border border-input',
    addon:   'px-3 py-2 bg-muted text-muted-foreground text-sm border-r border-input',
    input:   'flex-1 px-3 py-2 outline-none bg-transparent',
  },
  variants: {
    size: {
      sm: { root: 'text-xs', addon: 'px-2 py-1', input: 'px-2 py-1' },
      md: { root: 'text-sm', addon: 'px-3 py-2', input: 'px-3 py-2' },
    },
  },
});

const { root, addon, input } = inputGroupVariants({ size: 'md' });
```

Use single `tv()` (no slots) for simple single-element components like `Button`, `Badge`, `Kbd`.

## Tabs ‚Äî The Indicator + Height Animation Pattern

Tabs uses two separate mechanisms:

1. **`TabsPrimitive.Indicator`** ‚Äî Base UI writes `--active-tab-left`, `--active-tab-width`, `--active-tab-height`, `--active-tab-bottom` CSS custom properties to the DOM. The indicator uses `translate-x-(--active-tab-left)` + `w-(--active-tab-width)` with `transition: translate, width` + `will-change-[translate,width] transform-gpu`. Zero JS measurement.

2. **`TabsPanelsWrapper`** ‚Äî A `ResizeObserver` tracks content height, writes it to a `height` state, and applies it to the container with a CSS transition. This animates `height: auto` without JS animation libraries.

```tsx
// Indicator classes (segmented variant)
'-translate-y-(--active-tab-bottom) absolute bottom-0 left-0
 h-(--active-tab-height) w-(--active-tab-width) translate-x-(--active-tab-left)
 transition-[translate,width] duration-200 ease-[cubic-bezier(.25,.46,.45,.94)]
 will-change-[translate,width] transform-gpu'
```

## Accordion Swiss Variant ‚Äî CSS `has()` Pattern

The `swiss` variant expands an item into a floating card using zero JavaScript, purely via CSS `has()` and `data-open`:

```tsx
// Open item: fully rounded, colored border, margin
'data-open:rounded-2xl data-open:border data-open:border-primary/70 data-open:my-6'

// Item BEFORE an open item: get bottom rounded corners
'has-[+_[data-slot=accordion-item][data-open]]:rounded-b-2xl'

// Item AFTER an open item: get top rounded corners
'data-open:[&+[data-slot=accordion-item]]:rounded-t-2xl'
```

## Nested Dialog ‚Äî CSS Variable Stacking

Base UI writes `--nested-dialogs` (integer) to each dialog. Use CSS math to derive stacking offsets:

```css
/* Scale down per nesting level */
scale: calc(1 - 0.1 * var(--nested-dialogs));

/* Translate up per nesting level */
translate: 0 calc(-2rem * var(--nested-dialogs));
```

Unlimited nesting depth, zero JS tracking.
